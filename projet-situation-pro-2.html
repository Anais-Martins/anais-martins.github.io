Situation professionnelle #2 – Épreuve E6

Contexte

Date : 01/12/2025 au 04/04/2026

Cadre : École – IRIS Mediaschool Toulouse – Épreuve E6 du BTS SIO SLAM

Organisation support : École du cinéma de Toulouse (contexte fictif)

Modalité : Travail individuel

Dans la continuité du projet web (Situation pro #1), l'École du cinéma de Toulouse souhaitait disposer d'une application desktop permettant de visualiser des statistiques de gestion : taux de présence, moyennes des apprenants, répartition des notes. L'application consomme l'API REST du projet web via des appels HTTP.

Objectif

Développer une application desktop JavaFX en architecture client lourd (MVC + FXML), consommant l'API REST du projet 1, avec trois tableaux de bord statistiques adaptés selon le rôle de l'utilisateur connecté (Admin, Formateur, Apprenant).

Mon rôle

Développeuse desktop – Travail individuel. Conception et développement de l'application JavaFX, des services de consommation d'API et des tableaux de bord statistiques.

Réalisations

  • Développement de 3 tableaux de bord selon le rôle connecté, chacun avec son Controller FXML, son Service et ses modèles Java
  • Dashboard Admin : KPIs globaux (nombre d'apprenants, formateurs, formations, taux de présence global), graphique BarChart des inscrits par formation, PieChart de répartition des utilisateurs, tableau des dernières inscriptions
  • Dashboard Formateur : matière enseignée, nombre d'élèves suivis, moyenne de la promotion, PieChart de répartition des notes, tableau des cours planifiés, tableau des élèves avec moyenne et taux de présence individuels
  • Dashboard Apprenant : formation suivie, moyenne générale personnelle, taux de présence personnel, PieChart de répartition de ses notes, BarChart de sa moyenne par matière
  • Mise en place d'appels HTTP GET vers l'API REST (localhost:3001) via HttpURLConnection
  • Désérialisation des réponses JSON avec la bibliothèque Gson
  • Système de session (SessionUser) pour conserver les informations de l'utilisateur connecté entre les vues
  • Fenêtre de login avec authentification par rôle

Résultats

  • 3 dashboards opérationnels : tableaux de bord distincts et adaptés selon le rôle connecté (Admin, Formateur, Apprenant)
  • Affichage des KPIs en temps réel : données récupérées dynamiquement depuis l'API REST du projet 1
  • Graphiques générés dynamiquement : PieChart et BarChart alimentés par les données réelles (répartition des notes, inscrits par formation, moyennes par matière)
  • Tableau des élèves et statistiques : disponible pour le formateur avec moyenne et taux de présence individuels
  • Code source structuré et documenté : architecture MVC respectée, services séparés par rôle (StatAdminService, StatFormateurService, StatApprenantService)

Technologies

Java JavaFX 25 FXML Gson 2.13.1 Jackson 2.15.2 HttpURLConnection API REST MySQL phpMyAdmin XAMPP Visual Studio Code JDK 25

Compétences BTS SIO mobilisées

✓ Concevoir et développer une solution applicative

Développement d'une application desktop JavaFX en architecture MVC avec FXML, intégrant des graphiques dynamiques et une gestion de session.

✓ Assurer la maintenance corrective ou évolutive d'une solution applicative

Consommation et adaptation aux évolutions de l'API REST du projet 1, tests et débogage.

✓ Gérer les données

Désérialisation JSON, exploitation des données via l'API REST et affichage statistique dynamique (KPIs, graphiques, tableaux).
